# Архитектура приложения Data Sculptor

## Назначение

Приложение решает две файловые задачи:

1. Массовое переименование файлов по входной и выходной маске.
2. Конвертация данных из `.xlsx` в `.csv` с выбором листа, разделителя и режима обработки одного файла или папки.
3. Анализ и простое редактирование XLSX: проверка заполненных колонок и вставка новой колонки.
4. Потоковое редактирование CSV: проверка заголовков, вставка, удаление и swap колонок.

GUI использует `tkinter`, конвертация XLSX -> CSV использует стандартные модули `zipfile` и `xml.etree.ElementTree`, а редактирование XLSX выполняется через `openpyxl`. Редактирование вынесено на библиотеку, потому что ручная правка OOXML легко ломает реальные книги: кроме `sheetN.xml` нужно учитывать стили, таблицы, связи и другие части архива.

`openpyxl` загружается лениво: GUI стартует без обязательного импорта editor-backend. Если библиотека отсутствует, вкладка `XLSX editor` показывает понятный статус и просит вручную выполнить `pip install -r requirements.txt`.

## Общая структура

```text
core/
  __init__.py       публичный backend API пакета
  core.py           ядро массового переименования
  cli.py            CLI для переименования
  xlsx_csv.py       ядро XLSX -> CSV
  xlsx_cli.py       CLI для XLSX -> CSV
  xlsx_edit.py      ядро анализа и редактирования XLSX
  csv_edit.py       потоковое ядро редактирования CSV
  gui.py            desktop GUI на tkinter
tests/
  test_core.py      тесты переименования
  test_xlsx_csv.py  тесты XLSX -> CSV
data_sculptor_gui.pyw  Windows-запускатель GUI без консоли
README.md           пользовательская инструкция
```

Главная идея архитектуры: бизнес-логика не зависит от интерфейса. CLI и GUI вызывают одни и те же backend-функции, поэтому поведение остается одинаковым независимо от способа запуска.

## Слой переименования

Файл `core/core.py` отвечает за три этапа:

1. Компиляция входной маски в регулярное выражение.
2. Извлечение полей из имени файла и нормализация дат.
3. Построение и валидация плана переименования.

Пример:

```text
input:  {name} {date:DD.MM.YYYY}.{ext}
output: {date:YYYYMMDD}_{name}.{ext}
```

Файл `Report 31.05.2026.txt` превращается в `20260531_Report.txt`.

Важная деталь: сначала создается список `RenamePlan`, и только потом выполняется `apply_rename_plan`. Это снижает риск частичного переименования, потому что конфликты целевых имен проверяются заранее.

## Слой XLSX -> CSV

Файл `core/xlsx_csv.py` читает `.xlsx` как ZIP-архив. Внутри XLSX используются:

- `xl/workbook.xml` - список листов;
- `xl/_rels/workbook.xml.rels` - связь листа с XML-файлом;
- `xl/sharedStrings.xml` - общий словарь строк;
- `xl/worksheets/sheetN.xml` - данные конкретного листа.

Конвертация поддерживает:

- один файл `.xlsx` в один `.csv`;
- папку `.xlsx` файлов в папку `.csv`;
- рекурсивный обход вложенных папок;
- выбор листа по имени или по номеру;
- выбор разделителя CSV;
- автоматическое преобразование Excel serial dates в человекочитаемый формат;
- callback прогресса;
- мягкую отмену.

Для дат parser читает `xl/styles.xml`. Если ячейка хранит число, но ее стиль ссылается на date-format, значение преобразуется из Excel serial date. Например, `44953` становится `27.01.2023`. Обычные числовые ячейки без date-стиля остаются числами.

## Прогресс и отмена

GUI не должен выполнять долгую конвертацию в главном потоке. Иначе Windows помечает окно как `Not Responding`, потому что `tkinter` не успевает обрабатывать события и перерисовку.

Поэтому `core/gui.py` запускает конвертацию в `threading.Thread`. Worker-поток не обращается к tkinter напрямую. Он отправляет события в `queue.Queue`, а главный поток забирает их через `after()`.

Событие прогресса описано dataclass-классом `ConversionProgress`:

```text
source       текущий XLSX-файл
target       целевой CSV-файл
file_index   номер текущего файла
file_count   всего файлов
percent      прогресс текущего файла
message      стадия: Starting, Reading worksheet, Writing CSV, Done
```

Отмена работает через `threading.Event`. Кнопка `Cancel` выставляет флаг, backend проверяет его между крупными стадиями и перед каждым новым файлом. Это мягкая отмена: приложение не убивает поток принудительно и не обрывает системную запись файла посередине.

## GUI

GUI состоит из четырех вкладок:

- `Batch rename`;
- `XLSX to CSV`.
- `XLSX editor`.
- `CSV editor`.

Основные подписи GUI локализуются через словарь в `core/gui.py`. Меню `Language / Язык` переключает язык без перезапуска приложения и обновляет вкладки, кнопки, подписи форм и заголовки таблиц.

Вкладка переименования показывает preview-таблицу. Реальное действие выполняется только после подтверждения.

Вкладка XLSX позволяет выбрать:

- источник как файл или папку;
- цель как CSV-файл или папку;
- лист;
- разделитель;
- кодировку;
- рекурсивный режим;
- отмену во время конвертации.

После успешного действия или отмены показывается отдельное окно со статусом и кнопкой OK.

Вкладка `XLSX editor` вызывает backend из `xlsx_edit.py`. Она умеет быстро проверять колонки по первой строке заголовков в одном файле или папке XLSX, показывать букву колонки и заголовок, вставлять новую колонку в позицию `A`, `B`, `C`, `AA`, удалять выбранную колонку и менять местами две колонки. Управление вставкой, удалением и swap разделено на визуальные блоки, чтобы не смешивать смысл полей. Check columns использует быстрый OOXML-reader и не загружает всю книгу через `openpyxl`; вставка, удаление и swap редактируют XLSX-файлы на месте через `openpyxl`.

После проверки колонок GUI показывает таблицу файлов. Первая колонка таблицы работает как псевдо-чекбокс: `[x]` означает, что файл участвует в операции, `[ ]` означает пропуск. Клик по любой части строки переключает выбор. Общий чекбокс `Select all files` выбирает или снимает выбор со всех строк. Массовые операции вставки и удаления применяются только к отмеченным строкам. Если таблица еще не построена, операция применяется ко всему выбранному источнику.

Как и конвертация, XLSX editor выполняет долгие операции в фоновом потоке. Главный поток tkinter получает события через очередь и обновляет текущий файл, progress bar и статус. Кнопка `Cancel` останавливает обработку между файлами.

Вкладка `CSV editor` использует стандартный модуль `csv`, а не dataframe-библиотеку. Проверка читает только первую строку заголовков. Операции вставки, удаления и swap обрабатывают CSV построчно, записывают результат во временный файл и заменяют исходник только после успешного завершения.

## CLI

CLI-команды нужны для автоматизации и тестирования:

```powershell
python -m core.cli ...
python -m core.xlsx_cli ...
```

Обе команды тонкие: они разбирают аргументы, вызывают backend и печатают результат. В них нет отдельной бизнес-логики.

## Тестирование

Тесты используют стандартную библиотеку `unittest`; editor-тесты дополнительно используют `openpyxl`, потому что эта же библиотека отвечает за безопасное редактирование реальных XLSX.

Для тестов конвертера XLSX тесты вручную создают минимальный ZIP-архив с XML-файлами. Это дает два преимущества:

- тестируется именно наш парсер OOXML, а не поведение сторонней библиотеки.

Основные сценарии покрыты тестами:

- конвертация даты в имени файла;
- перестановка имени и даты;
- конфликт целевых имен;
- фактическое применение плана переименования;
- чтение XLSX-листов;
- shared strings и inline strings;
- boolean-значения;
- конвертация папки;
- рекурсивная конвертация;
- события прогресса;
- отмена конвертации.
- проверка заполненных XLSX-колонок;
- вставка XLSX-колонки в один файл и в папку файлов.
- удаление XLSX-колонки из одного файла и из папки файлов.
- swap двух XLSX-колонок в одном файле и в папке файлов.

## Ограничения текущей версии

Прогресс внутри одного XLSX сейчас считается по стадиям, а не по реальному количеству строк. Это сделано потому, что текущий парсер читает XML листа целиком. Для построчного прогресса нужно перейти на потоковый XML-разбор через `ElementTree.iterparse`.

Excel-даты поддерживаются для стандартной date system 1900 и для ячеек, где дата задана через стиль. Если дата записана как обычное число без date-стиля, приложение не может надежно отличить ее от настоящего числа и оставляет значение как есть.

## Возможные следующие шаги

- Построчный streaming-парсер XLSX для точного прогресса по строкам.
- Настройка формата Excel-дат при выгрузке.
- Сохранение последних выбранных папок и настроек GUI.
- Упаковка приложения в `.exe` через PyInstaller.
